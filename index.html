🔬 核心技术原理

综合 Caveman 项目 + TOP10 Token 节省方法的研究成果

🪨 输出风格压缩

通过 System Prompt 工程改变模型输出风格，让模型从源头少生成低信息密度的自然语言。技术术语、代码、URL 保持逐字节不变。

来源：Caveman 核心原理

📦 输入上下文压缩

自动压缩规则文件、记忆文件等上下文，去除冗余描述和填充词。代码块、路径、URL、技术术语逐字节保护。

来源：caveman-compress + LLMLingua

🔒 安全兜底机制

在安全警告、不可逆操作确认、多步骤复杂操作、用户困惑等场景自动退出压缩模式，恢复正常清晰表达。

来源：Caveman Auto-Clarity 机制

📜 中文文言文优化

利用文言文天然的高信息密度（单字承载更多语义），在中文场景实现比英文 Caveman 更深层的压缩。

来源：caveman-cn 中文适配

🏗️ 结构化输出引导

引导模型使用 JSON、列表、表格等结构化格式，减少冗余自然语言描述，提高信息密度。

来源：TOP10 方法 #4, #9, #10

📊 Token 实时监控

精确统计输入/输出 Token 数量，支持多种模型的 tokenizer 近似，提供压缩前后对比分析。

来源：TOP10 方法 #10 监控分析

🔑 关键发现与最佳实践

基于 Benchmark 测试的核心结论

✅ 发现 1：多轮对话是收益核心

System Prompt 是一次性开销（约350 Token），在单轮对话中优势不明显（3.4%），但从第5轮开始节省率跃升至 53%+。WorkBuddy 的典型使用场景正是长对话。

✅ 发现 2：五大模型效果一致

DeepSeek、GLM、MiniMax、Hunyuan、Kimi 五个国产大模型在使用 Token Saver 后均获得一致的节省效果（59.7%），因为压缩作用于语言层面而非模型推理层面。

✅ 发现 3：文言文模式对中文有额外优势

文言文的天然高信息密度使 Ultra 模式达到 64.5% 的节省率，这是中文独有的优化空间。英文 Caveman 的文言文模式不存在此优势。

⚠️ 注意：短对话不适用

对于 1-2 轮的超短对话，System Prompt 开销可能抵消收益。建议：长对话使用 Full/Ultra 模式，短对话使用 Lite 或不用。

💡 使用建议

📋 场景 1：日常编码助手

推荐：Full 模式
你的 WorkBuddy 代码助手通常是长时间多轮对话，Full 模式在保持技术准确性的同时节省约 60% Token。

🔍 场景 2：代码审查

推荐：Ultra 模式
代码审查需要快速指出问题，Ultra 模式的电报体格式完美契合这一场景。

📚 场景 3：技术文档阅读

推荐：Wenyan 模式
中文技术文档的总结和翻译场景中，文言文模式可以在保持简洁的同时保留技术精确性。

🆕 场景 4：快速问答

推荐：Lite 模式
简单的技术问答不需要过度压缩，Lite 模式仅去除填充词，保持自然对话感。
